# 점프서버(192.168.13.126) → 192.168.1.146 SSH 안 되는 이유 조사

## 리포에서 확인한 사실

이 GitOps 리포에서 **192.168.1.146**은 **호스트 IP가 아니라 MetalLB/게이트웨이용 VIP**로 쓰입니다.

| 용도 | 파일 | 내용 |
|------|------|------|
| MetalLB 풀 주소 | `clusters/onprem/dev/platform/metallb-system/values.yaml` | `addresses: - 192.168.1.146/32` |
| Envoy Gateway LoadBalancerIP | `clusters/onprem/dev/platform/gateway/values.yaml` | `loadBalancerIP: 192.168.1.146` |
| ips.env | `scripts/ips.env` | `ONPREM_GATEWAY_LB_IP=192.168.1.146` |

즉 **192.168.1.146**은 **Envoy Gateway LoadBalancer Service**에 할당된 **MetalLB 가상 IP**입니다.  
그 IP로 들어오는 트래픽은 **Service가 열어둔 포트(80, 443 등)**만 노드로 전달되고, **22번(SSH)는 해당 Service에 없어서** 점프서버에서 `ssh 192.168.1.146`이 실패할 수 있습니다.

반면 **192.168.1.147**, **192.168.1.148**은 리포에서 **노드/호스트 IP**로 쓰입니다(Loki, NFS, fluent-bit 등).  
실제 노드(또는 VM)의 주소라서 SSH가 열려 있는 것이 자연스럽습니다.

---

## 결론과 대안

- **192.168.1.146만 SSH가 안 되는 이유:**  
  그 주소가 **MetalLB가 게이트웨이용으로 쓴 VIP**라서, 22번 포트는 서비스에 없고 80/443 등만 열려 있기 때문일 가능성이 큽니다.
- **Argo CD/포트포워드는 어디서 띄우나:**  
  Argo CD·kubectl을 실행하는 건 **실제 노드(eks-master 등)**이므로, 그 노드의 **진짜 호스트 IP**는 192.168.1.147 또는 192.168.1.148 중 하나일 가능성이 높습니다.
- **점프서버에서 할 일:**  
  - Argo CD UI 접속용: **Argo CD·포트포워드를 띄운 그 호스트의 실제 IP**(147 또는 148)로 SSH 접속한 뒤,  
    그 호스트에서 `kubectl port-forward --address 0.0.0.0 ...` 을 띄우고,  
    점프서버에서는 `ssh -L 8080:<그_호스트_IP>:8080 xubuntu@192.168.13.126` 로 터널 연결.
  - 또는 **147/148 중 “마스터/Argo CD 호스트”에 SSH**해서, 그 위에서 포트포워드 + 로컬에서 `ssh -L 8080:localhost:8080 -J xubuntu@192.168.13.126 <user>@<147 또는 148>` 로 접속.

## 점프서버에서 바로 확인하는 방법

점프서버(192.168.13.126)에 로그인한 뒤 아래처럼 비교해 보세요.

```bash
# 이 리포의 스크립트를 점프서버로 복사해서 실행하거나,
# 점프서버에서 직접:
nc -zv 192.168.1.146 22   # SSH → 실패 예상
nc -zv 192.168.1.146 80   # Gateway HTTP → 성공 가능
nc -zv 192.168.1.147 22   # SSH → 성공 예상
nc -zv 192.168.1.148 22   # SSH → 성공 예상
```

또는 리포 루트에서:

```bash
ssh xubuntu@192.168.13.126 "bash -s" < scripts/check-jump-to-1dot146.sh
```

(점프서버 인증이 되는 환경에서만 실행 가능)

## 정리

| IP | 이 리포에서의 역할 | 점프서버에서 SSH |
|----|--------------------|------------------|
| 192.168.1.146 | MetalLB VIP, Gateway LoadBalancer | 22번 없음 → **실패** |
| 192.168.1.147 | Loki/NFS/노드 등 호스트 IP | **가능** |
| 192.168.1.148 | 노드/호스트 IP | **가능** |

Argo CD를 쓰는 **실제 노드가 147인지 148인지**만 확인한 뒤, 그 IP로 SSH하고 그 위에서 포트포워드·터널을 연결하면 됩니다.
